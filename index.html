Swift Basics

Variables · Optionals · Functions · Closures
What it is

Swift is Apple's compiled, type-safe programming language introduced in 2014 as a modern replacement for Objective-C. It is used to build apps for iOS, macOS, watchOS, and tvOS. Swift's standout features are optionals (values that may or may not exist), type inference (the compiler figures out types from context), closures (self-contained blocks of code you can pass around), and clean syntax that reduces boilerplate. Every iOS project starts here — models, logic, views, and services are all written in Swift.

Code Snippet
swift 
// Constants and variables
let appName: String = "CardVault"    // immutable
var cardCount: Int = 0              // mutable

// Type inference — compiler deduces the type
var marketValue = 45.00             // Double
var isGraded = false                // Bool

// Optionals — a value that might be nil
var cardOwner: String? = nil
cardOwner = "neder_dev"

// Safe unwrapping — guard lets you exit early
func greetOwner(_ name: String?) -> String {
    guard let name = name else {
        return "Welcome, guest collector!"
    }
    return "Vault accessed by: \(name)!"
}

// nil coalescing — provide a fallback
let display = cardOwner ?? "Anonymous Collector"   // "neder_dev"

// Closure — a block you can assign and pass
let doubleValue: (Int) -> Int = { v in v * 2 }
print(doubleValue(50))   // 100

// Trailing closure syntax (used everywhere in SwiftUI)
let prices = [12.5, 250.0, 4.75, 120.0]
let premiumCards = prices.filter { $0 > 50.0 }
print(premiumCards)   // [250.0, 120.0]

// async/await — modern concurrency
func fetchCardData() async throws -> Data {
    let url = URL(string: "https://api.example.com/cards")!
    let (data, _) = try await URLSession.shared.data(from: url)
    return data
}
🃏
Real Use Case

Every line of my card collecting app is Swift. Optionals handled missing fields in the JSON (like a card illustration URL or grading subgrades that might be absent), closures powered button callbacks and sorting parameters, and async/await made network calls to the trading card database API clean and readable without nested callbacks.

OOP — Classes, Structs, Enums & Protocols

Value types · Reference types · Named cases · Contracts
What it is

Swift has four building blocks for organizing data and behavior. Structs are value types — copied when assigned or passed, making them thread-safe and predictable by default. Classes are reference types — all variables point to the same object in memory, ideal for shared mutable state. Enums define a fixed, exhaustive set of named cases with optional associated values. Protocols define a contract — a required set of properties and methods — that any type can adopt, enabling polymorphism and dependency injection without rigid class hierarchies.

Code Snippet
swift 
// Protocol — a contract any type can adopt
protocol Displayable {
    var displayName: String { get }
    func describe() -> String
}

// Enum — fixed named cases (with raw value)
enum CardType: String, CaseIterable {
    case creature = "Creature"
    case spell    = "Spell"
    case energy   = "Energy"
}

// Struct — value type, conforms to protocols
struct TradingCard: Displayable, Identifiable, Codable {
    let id: UUID
    var name: String
    var marketPrice: Double
    var type: CardType

    // Computed property
    var displayName: String { name }
    func describe() -> String {
        "\(name) (\(type.rawValue)) — $\(marketPrice)"
    }
}

// Class — reference type, shared mutable state
class CardBinder {
    private(set) var cards: [TradingCard] = []

    func add(_ card: TradingCard) {
        guard !cards.contains(where: { $0.id == card.id }) else { return }
        cards.append(card)
    }

    func remove(_ card: TradingCard) {
        cards.removeAll { $0.id == card.id }
    }

    var totalPortfolioValue: Double {
        cards.reduce(0.0) { $0 + $1.marketPrice }
    }
}

// Usage
let rareMonster = TradingCard(id: UUID(), name: "Cyber Dragon",
                              marketPrice: 89.99, type: .creature)
let binder = CardBinder()
binder.add(rareMonster)
print(binder.totalPortfolioValue)     // 89.99
print(rareMonster.describe())         // Cyber Dragon (Creature) — $89.99

// Struct is copied — binder portfolio is NOT affected
var cardCopy = rareMonster
cardCopy.marketPrice = 9999.0
print(binder.totalPortfolioValue)     // still 89.99
🏗️
Real Use Case

In my card collecting app, TradingCard was a struct (safe to copy and display inside UI cells), while CardBinder was a class (one shared inventory instance passed throughout the app). CardType was an enum so SwiftUI's ForEach could cleanly iterate all cases to build type-based filter tabs. Identifiable and Codable were protocols required by SwiftUI lists and JSON decoders automatically.

03

SwiftUI — Views, Layouts & Modifiers

Declarative UI · Stacks · State · Composition
What it is

SwiftUI is Apple's declarative UI framework, released in 2019. Instead of imperatively creating and configuring views one by one (like UIKit), you declare what the UI looks like given a certain state — and SwiftUI re-renders the affected parts automatically when that state changes. Layouts are built by composing stacks: VStack (vertical), HStack (horizontal), and ZStack (layered). Modifiers chain off any view to customize font, color, padding, shadow, animation, and more — each modifier returns a new, wrapped view so they compose cleanly.

Code Snippet
swift 
import SwiftUI

struct TradingCardView: View {
    let card: TradingCard
    @State private var isWishlisted = false

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {

            // Hero artwork area
            ZStack(alignment: .topTrailing) {
                RoundedRectangle(cornerRadius: 14)
                    .fill(Color(.systemGray6))
                    .frame(height: 180)

                Image(systemName: card.type == .creature
                      ? "shield.fill" : "wand.and.stars")
                    .font(.system(size: 44))
                    .foregroundStyle(.secondary)
                    .frame(maxWidth: .infinity, maxHeight: .infinity)

                // Rarity / Type badge
                Text(card.type.rawValue)
                    .font(.caption2.weight(.semibold))
                    .padding(.horizontal, 8).padding(.vertical, 4)
                    .background(.ultraThinMaterial, in: Capsule())
                    .padding(10)
            }

            // Text block
            VStack(alignment: .leading, spacing: 3) {
                Text(card.name)
                    .font(.headline)
                    .foregroundStyle(.primary)
                Text(card.describe())
                    .font(.caption)
                    .foregroundStyle(.secondary)
                    .lineLimit(1)
            }

            // Price + action row
            HStack(alignment: .center) {
                Text("$\(card.marketPrice, specifier: "%.2f")")
                    .font(.title3.weight(.bold))
                    .foregroundStyle(.blue)

                Spacer()

                Button {
                    withAnimation(.spring(response: 0.3)) {
                        isWishlisted.toggle()
                    }
                } label: {
                    Label(
                        isWishlisted ? "Saved" : "Wishlist",
                        systemImage: isWishlisted
                            ? "heart.fill"
                            : "heart"
                    )
                    .font(.subheadline.weight(.medium))
                    .foregroundStyle(isWishlisted ? .red : .blue)
                }
                .buttonStyle(.plain)
            }
        }
        .padding(16)
        .background(Color(.systemBackground))
        .clipShape(RoundedRectangle(cornerRadius: 18))
        .shadow(color: .black.opacity(0.09), radius: 10, y: 5)
    }
}
📱
Real Use Case

Every screen in my card collecting app was built with SwiftUI. The main showcase screen used a LazyVGrid of TradingCardViews. The card detail view used a ScrollView with a ZStack for displaying holographic overlays. All styling was done through modifier chains — no Interface Builder, no storyboards. @State on isWishlisted caused the heart button to re-render automatically with the appropriate icon fill and red tint on every tap.

04

MVVM + @Observable

Architecture · Separation of concerns · Reactive state
What it is

Model-View-ViewModel (MVVM) is an architecture pattern that separates code into three layers. The Model holds raw data. The ViewModel contains business logic, transforms data for display, and handles async work. The View simply renders what the ViewModel exposes — no logic lives there. In iOS 17+, the @Observable macro replaces the older @ObservableObject + @Published pattern. It automatically tracks which properties each View reads and triggers UI updates only when those specific values change — more granular and more efficient.

Code Snippet
swift 
import SwiftUI
import Observation

// ── ViewModel — all business logic lives here ──────────
@Observable
class BinderViewModel {
    var cards:     [TradingCard] = []
    var isLoading: Bool          = false
    var errorMessage: String?

    // Derived state — computed from cards
    var totalValue: Double { cards.reduce(0) { $0 + $1.marketPrice } }
    var cardCount:  Int    { cards.count }
    var isEmpty:    Bool   { cards.isEmpty }

    func add(_ card: TradingCard) {
        guard !cards.contains(where: { $0.id == card.id }) else { return }
        cards.append(card)
    }

    func remove(_ card: TradingCard) {
        cards.removeAll { $0.id == card.id }
    }

    func clearBinder() { cards.removeAll() }

    func syncWithCloud() async {
        isLoading = true
        defer { isLoading = false }
        do {
            let collectionData = CollectionPayload(cards: cards)
            let _ = try await CardAPIService.syncCollection(collectionData)
        } catch {
            errorMessage = error.localizedDescription
        }
    }
}

// ── View — only renders, zero business logic ───────────
struct BinderView: View {
    // @State with @Observable (no @StateObject needed)
    @State private var vm = BinderViewModel()

    var body: some View {
        NavigationStack {
            Group {
                if vm.isEmpty {
                    ContentUnavailableView(
                        "Your binder is empty",
                        systemImage: "square.stack.3d.up",
                        description: Text("Scan physical cards or browse the database to add them")
                    )
                } else {
                    List {
                        ForEach(vm.cards) { card in
                            HStack {
                                Text(card.name)
                                Spacer()
                                Text("$\(card.marketPrice, specifier: "%.2f")")
                                    .foregroundStyle(.secondary)
                            }
                            .swipeActions {
                                Button("Delete", role: .destructive) {
                                    vm.remove(card)
                                }
                            }
                        }

                        Section {
                            HStack {
                                Text("Total Portfolio Value").bold()
                                Spacer()
                                Text("$\(vm.totalValue, specifier: "%.2f")").bold()
                            }
                        }
                    }
                }
            }
            .navigationTitle("My Binder (\(vm.cardCount))")
            .toolbar {
                ToolbarItem(placement: .bottomBar) {
                    Button("Sync Collection") { Task { await vm.syncWithCloud() } }
                        .disabled(vm.isEmpty)
                        .buttonStyle(.borderedProminent)
                }
            }
            .overlay { if vm.isLoading { ProgressView() } }
        }
    }
}
🔁
Real Use Case

BinderViewModel handled all add/remove logic, portfolio value calculations, and the async cloud synchronization request — none of that structural math lived in the View layer. BinderView just observed properties from the view model and rendered UI updates accordingly. This separation made it incredibly simple to test inventory computations detached from visual layouts.

Navigation — NavigationStack & TabView

Push-pop hierarchy · Tab bars · Deep linking
What it is

NavigationStack (iOS 16+) manages a push/pop hierarchy of views. The user drills in by tapping a NavigationLink and goes back with the system back button. It uses a type-safe, value-based API: you attach a navigationDestination modifier that maps a value type to a destination view. TabView creates a tab bar at the bottom of the screen — each tab is an independent root view with its own navigation stack. Together, these two patterns cover nearly every navigation need in a standard iOS app.

Code Snippet
swift 
import SwiftUI

// ── Root: TabView with three sections ─────────────────
struct AppRootView: View {
    @State private var selectedTab = 0

    var body: some View {
        TabView(selection: $selectedTab) {
            CollectionTab()
                .tabItem { Label("Collection", systemImage: "square.stack.fill") }
                .tag(0)

            WishlistTab()
                .tabItem { Label("Wishlist",   systemImage: "heart.fill") }
                .tag(1)

            ProfileTab()
                .tabItem { Label("Profile",    systemImage: "person.circle") }
                .tag(2)
        }
    }
}

// ── Collection tab — NavigationStack for drill-down ───
struct CollectionTab: View {
    let inventory: [TradingCard] = TradingCard.samples

    var body: some View {
        NavigationStack {
            List(inventory) { card in
                // Value-based link — pushes when tapped
                NavigationLink(value: card) {
                    CardRowView(card: card)
                }
            }
            .navigationTitle("Main Vault")
            .navigationBarTitleDisplayMode(.large)
            // Maps a TradingCard value → a destination view
            .navigationDestination(for: TradingCard.self) { card in
                CardDetailView(card: card)
            }
        }
    }
}

// ── Programmatic navigation with a path ───────────────
struct CollectionWithPathView: View {
    @State private var navigationPath = NavigationPath()

    var body: some View {
        NavigationStack(path: $navigationPath) {
            VStack(spacing: 16) {
                Button("Inspect Secret Rare Card") {
                    navigationPath.append(TradingCard.exampleRare)
                }
                Button("Return to Library Root") {
                    navigationPath.removeLast(navigationPath.count)
                }
            }
            .navigationTitle("Vault Dashboard")
            .navigationDestination(for: TradingCard.self) { c in
                CardDetailView(card: c)
            }
        }
    }
}
🗺️
Real Use Case

My card collecting app used a TabView configuration containing three core views: Collection, Wishlist, and Profile. The Collection tab embedded a value-driven NavigationStack, ensuring that tapping individual cards safely pushed a dedicated CardDetailView to inspect grading details and market history. Value-based navigation states make deep-linking direct card scans effortless.

Lists & Tables

Lazy rows · Swipe actions · Sections · Search
What it is

List is SwiftUI's primary scrollable container for displaying rows of data. It renders lazily — only visible rows are created, making it efficient even with thousands of items. List supports swipe actions, pull-to-refresh, editing mode, multi-selection, and sections. The .searchable modifier adds a system search bar that filters the data source in real time. For data-grid-style layouts on iPad and macOS, Table provides sortable columns with a header row. List and Table cover virtually every data-display scenario in iOS apps.

Code Snippet
swift 
import SwiftUI

struct CardBinderListView: View {
    @State private var allCards: [TradingCard] = TradingCard.samples
    @State private var searchText = ""

    // Computed: filtered by search text, grouped by type
    var filteredCards: [TradingCard] {
        searchText.isEmpty
            ? allCards
            : allCards.filter {
                $0.name.localizedCaseInsensitiveContains(searchText)
              }
    }

    var groupedByTypes: [CardType: [TradingCard]] {
        Dictionary(grouping: filteredCards) { $0.type }
    }

    var body: some View {
        List {
            ForEach(CardType.allCases, id: \.self) { typeKey in
                if let sections = groupedByTypes[typeKey], !sections.isEmpty {
                    Section(typeKey.rawValue) {
                        ForEach(sections) { card in
                            CardRowView(card: card)
                                // Swipe left — destructive delete from binder
                                .swipeActions(edge: .trailing) {
                                    Button("Delete", role: .destructive) {
                                        deleteCard(card)
                                    }
                                }
                                // Swipe right — quick add to wishlist
                                .swipeActions(edge: .leading) {
                                    Button {
                                        addToWishlist(card)
                                    } label: {
                                        Label("Wishlist", systemImage: "heart.badge.plus")
                                    }
                                    .tint(.green)
                                }
                        }
                    }
                }
            }
        }
        .listStyle(.insetGrouped)
        .searchable(text: $searchText, prompt: "Search index matching name…")
        .refreshable {
            // Pull-to-refresh — async values update
            await reloadLiveMarketPrices()
        }
        .animation(.default, value: filteredCards)
        .navigationTitle("Card Index")
    }

    func deleteCard(_ c: TradingCard)      { allCards.removeAll { $0.id == c.id } }
    func addToWishlist(_ c: TradingCard)   { /* Append to wishlist data controller */ }
    func reloadLiveMarketPrices() async    { /* await API refresh loop */ }
}

Left-swipe shortcuts dropped items straight into a deck planner, while right swipes purged items from local collections. WebView & HTTP — REST / JSON

URLSession · Codable · async/await · WKWebView
What it is

Most iOS apps talk to a backend via HTTP REST APIs. Apple's URLSession handles all network requests; with Swift's async/await, calls are linear and readable. JSON responses map to Swift types automatically using the Codable protocol — just conform your model and JSONDecoder handles the conversion. POST requests send encoded Swift types as JSON bodies. WKWebView (wrapped in a UIViewRepresentable) embeds a full web browser inside a native SwiftUI view, useful for terms pages, external content, or hybrid web features.

Code Snippet
swift 
import SwiftUI
import WebKit

// ── 1. Codable model — maps JSON keys automatically ───
struct TradingCard: Codable, Identifiable {
    let id:          Int
    let name:        String
    let marketPrice: Double
    let type:        String
    let imageURL:    String?

    enum CodingKeys: String, CodingKey {
        case id, name, type
        case marketPrice = "market_price" // snake_case → camelCase
        case imageURL    = "image_url"
    }
}

// ── 2. Network service layer ──────────────────────────
struct CardAPIService {
    private static let baseURL = "https://api.cardvault.com/v1"

    // GET — decode global registry database array
    static func fetchCardRegistry() async throws -> [TradingCard] {
        let url = URL(string: "\(baseURL)/cards")!
        let (data, response) = try await URLSession.shared.data(from: url)
        guard (response as? HTTPURLResponse)?.statusCode == 200
        else { throw URLError(.badServerResponse) }
        return try JSONDecoder().decode([TradingCard].self, from: data)
    }

    // POST — upload local collection contents to server registry
    static func syncCollection(_ payload: CollectionPayload) async throws -> SyncReceipt {
        var request = URLRequest(url: URL(string: "\(baseURL)/sync")!)
        request.httpMethod = "POST"
        request.setValue("application/json", forHTTPHeaderField: "Content-Type")
        request.httpBody = try JSONEncoder().encode(payload)
        let (data, _) = try await URLSession.shared.data(for: request)
        return try JSONDecoder().decode(SyncReceipt.self, from: data)
    }
}

// ── 3. WKWebView in SwiftUI (Used for official pricing rulebooks)
struct TCGWebView: UIViewRepresentable {
    let siteURL: URL

    func makeUIView(context: Context) -> WKWebView {
        let webView = WKWebView()
        webView.load(URLRequest(url: siteURL))
        return webView
    }
    func updateUIView(_ uiView: WKWebView, context: Context) {}
}

// ── 4. View that fetches on appear ────────────────────
struct DatabaseView: View {
    @State private var networkCards: [TradingCard] = []
    @State private var itemsLoading = false

    var body: some View {
        List(networkCards) { card in Text(card.name) }
            .overlay { if itemsLoading { ProgressView() } }
            .task {
                itemsLoading = true
                defer { itemsLoading = false }
                networkCards = (try? await CardAPIService.fetchCardRegistry()) ?? []
            }
    }
}

Maps — MapKit

Interactive maps · Markers · Annotations · Routes
What it is

MapKit is Apple's framework for embedding interactive maps in iOS apps. The SwiftUI Map view (iOS 17+ API) accepts a MapCameraPosition binding that controls the viewport, and displays Marker and Annotation views at geographic coordinates. You can show the user's real-time location with UserAnnotation, draw route overlays with MapPolyline, and respond to map tap and item-selection events. MapKit tiles are powered by Apple Maps and automatically adapt to light and dark mode.

Code Snippet
swift 
import SwiftUI
import MapKit

struct CardShopLocation: Identifiable {
    let id = UUID()
    let shopName:   String
    let addressName: String
    let coordinates: CLLocationCoordinate2D
}

struct CardShopMapView: View {
    let retailOutlets: [CardShopLocation] = [
        CardShopLocation(shopName: "Hobby Central Cards",
                         addressName: "Downtown, Chihuahua",
                         coordinates: .init(latitude: 28.6353, longitude: -106.0889)),
        CardShopLocation(shopName: "Collector Cave",
                         addressName: "North Zone, Chihuahua",
                         coordinates: .init(latitude: 28.6500, longitude: -106.1050))
    ]

    @State private var positionFocus: MapCameraPosition = .automatic
    @State private var selectedShopID: UUID?

    var currentSelectedShop: CardShopLocation? {
        retailOutlets.first { $0.id == selectedShopID }
    }

    var body: some View {
        Map(position: $positionFocus, selection: $selectedShopID) {

            // Device GPS anchor indicator
            UserAnnotation()

            // Map pin entries for hobby stores
            ForEach(retailOutlets) { shop in
                Marker(shop.shopName, coordinate: shop.coordinates)
                    .tint(.blue)
                    .tag(shop.id)
            }
        }
        .mapControls {
            MapCompass()
            MapUserLocationButton()
            MapScaleView()
        }
        .mapStyle(.standard(elevation: .realistic))
        // Event sheet summary details layout overlay toggle
        .safeAreaInset(edge: .bottom) {
            if let shop = currentSelectedShop {
                HStack {
                    VStack(alignment: .leading, spacing: 4) {
                        Text(shop.shopName).font(.headline)
                        Text(shop.addressName).font(.subheadline)
                            .foregroundStyle(.secondary)
                    }
                    Spacer()
                    Button("Plot Route") {
                        openMapsAppRoute(to: shop)
                    }
                    .buttonStyle(.bordered)
                }
                .padding()
                .background(.regularMaterial)
            }
        }
        .ignoresSafeArea(edges: .top)
    }

    func openMapsAppRoute(to shop: CardShopLocation) {
        let mapItem = MKMapItem(placemark: MKPlacemark(coordinate: shop.coordinates))
        mapItem.name = shop.shopName
        mapItem.openInMaps()
    }
}

Camera & QR Reader

AVFoundation · AVCaptureSession · UIViewRepresentable
What it is

iOS camera access is managed by AVFoundation. An AVCaptureSession wires together inputs (the camera device) and outputs (still images, video, or metadata). For QR reading, you add an AVCaptureMetadataOutput — the framework fires a delegate method every time a QR code or barcode is detected in the live camera frame. Because AVFoundation is a UIKit API, it must be wrapped in a UIViewRepresentable to bridge into SwiftUI. The camera permission key NSCameraUsageDescription must be added to Info.plist.

Code Snippet
swift 
import SwiftUI
import AVFoundation

// ── UIViewRepresentable bridges AVFoundation → SwiftUI ─
struct CardScannerView: UIViewRepresentable {
    var onTokenDetected: (String) -> Void

    // Coordinator acts as the AVFoundation delegate
    class Coordinator: NSObject, AVCaptureMetadataOutputObjectsDelegate {
        let targetView: CardScannerView
        init(_ viewInstance: CardScannerView) { self.targetView = viewInstance }

        func metadataOutput(
            _ output: AVCaptureMetadataOutput,
            didOutput objects: [AVMetadataObject],
            from connection: AVCaptureConnection
        ) {
            guard
                let rawObj = objects.first as? AVMetadataMachineReadableCodeObject,
                let valueString = rawObj.stringValue
            else { return }

            // Ensure UI actions run safely on the primary loop thread
            DispatchQueue.main.async {
                self.targetView.onTokenDetected(valueString)
            }
        }
    }

    func makeCoordinator() -> Coordinator { Coordinator(self) }

    func makeUIView(context: Context) -> UIView {
        let view          = UIView(frame: .zero)
        let captureStream = AVCaptureSession()

        guard
            let cameraSource = AVCaptureDevice.default(for: .video),
            let captureInput = try? AVCaptureDeviceInput(device: cameraSource),
            captureStream.canAddInput(captureInput)
        else { return view }

        captureStream.addInput(captureInput)

        let metadataPipeline = AVCaptureMetadataOutput()
        captureStream.addOutput(metadataPipeline)
        metadataPipeline.setMetadataObjectsDelegate(context.coordinator, queue: .main)
        metadataPipeline.metadataObjectTypes = [.qr, .code128, .ean13]

        // Video configuration layer context linkage
        let renderingLayer = AVCaptureVideoPreviewLayer(session: captureStream)
        renderingLayer.videoGravity = .resizeAspectFill
        renderingLayer.frame = UIScreen.main.bounds
        view.layer.addSublayer(renderingLayer)

        DispatchQueue.global(qos: .userInitiated).async {
            captureStream.startRunning()
        }
        return view
    }

    func updateUIView(_ uiView: UIView, context: Context) {}
}

// ── SwiftUI screen that uses the scanner ──────────────
struct InventoryScannerScreen: View {
    @State private var registeredCode: String?
    @State private var displayAlert = false

    var body: some View {
        ZStack {

            }
            .ignoresSafeArea()

                .stroke(.white, lineWidth: 2)

        }
        .alert("Code Successfully Scanned", isPresented: $displayAlert) {
            Button("Dismiss") {}
        } message: {
            Text("Decoded Token Data ID: \(registeredCode ?? "Unknown Card")")

    }
}

CoreLocation

GPS · Permissions · CLLocation · Distance
What it is

CoreLocation gives iOS apps access to the device's geographic position using GPS, Wi-Fi triangulation, and cellular data. Before receiving any location, you must request user permission — either when in use (only while the app is open) or always (including when backgrounded). Updates are delivered via CLLocationManagerDelegate callbacks. You can compute the distance between two CLLocation objects with a single method call, or set up geofences (CLCircularRegion) that fire events when the user enters or exits an area. The required Info.plist key is NSLocationWhenInUseUsageDescription.

Code Snippet
swift 
import CoreLocation
import SwiftUI

// ── Observable wrapper around CLLocationManager ────────
@Observable
class UserLocationManager: NSObject, CLLocationManagerDelegate {
    private let coreLocationService = CLLocationManager()
    var lastRecordedLocation: CLLocation?
    var permissionStatus: CLAuthorizationStatus = .notDetermined

    override init() {
        super.init()
        coreLocationService.delegate = self
        coreLocationService.desiredAccuracy = kCLLocationAccuracyNearestTenMeters
    }

    func requestDevicePermission() {
        coreLocationService.requestWhenInUseAuthorization()
    }

    // Delegate — called when permission changes
    func locationManagerDidChangeAuthorization(_ manager: CLLocationManager) {
        permissionStatus = manager.authorizationStatus
        if permissionStatus == .authorizedWhenInUse || permissionStatus == .authorizedAlways {
            coreLocationService.startUpdatingLocation()
        }
    }

    // Delegate — called with new GPS fix
    func locationManager(_ manager: CLLocationManager, didUpdateLocations positions: [CLLocation]) {
        lastRecordedLocation = positions.last
    }

    func locationManager(_ manager: CLLocationManager, didFailWithError error: Error) {
        print("GPS Stream Error: \(error.localizedDescription)")
    }

    // Utility: distance calculation metric output in meters
    func measureMetresToTarget(targetCoords: CLLocationCoordinate2D) -> Double? {
        guard let currentAnchor = lastRecordedLocation else { return nil }
        let targetSpot = CLLocation(latitude: targetCoords.latitude, longitude: targetCoords.longitude)
        return currentAnchor.distance(from: targetSpot)
    }
}

// ── View: show nearest trade meetup store ─────────────
struct ClosestStoreView: View {
    @State private var locationService = UserLocationManager()
    let retailNetworks = CardShopLocation.samples

    var closestShop: CardShopLocation? {
        retailNetworks.min {
            (locationService.measureMetresToTarget(targetCoords: $0.coordinates) ?? .infinity) <
            (locationService.measureMetresToTarget(targetCoords: $1.coordinates) ?? .infinity)
        }
    }

    var body: some View {
        VStack(spacing: 24) {
            if locationService.permissionStatus == .notDetermined {
                Button("Authorize Location Tracking") {
                    locationService.requestDevicePermission()
                }
                .buttonStyle(.borderedProminent)
            } else if let shop = closestShop,
                      let rangeDist = locationService.measureMetresToTarget(targetCoords: shop.coordinates) {
                VStack(spacing: 6) {
                    Text("Closest Local Trading Spot").font(.caption).foregroundStyle(.secondary)
                    Text(shop.shopName).font(.title2.bold())
                    Text(String(format: "%.0f meters from current location", rangeDist))
                        .foregroundStyle(.secondary)
                }
            } else {
                ProgressView("Calibrating device GPS metrics…")
            }
        }
        .onAppear { locationService.requestDevicePermission() }
        .padding()
    }
}
